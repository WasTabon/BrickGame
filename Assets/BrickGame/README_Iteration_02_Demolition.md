# BrickGame — Iteration 02: Demolition Scene (Core Physics)

## 1. Что было добавлено
Ядро механики на реальной 2D-физике: машина слева, башня из физических кирпичей (опоры, кирпичи, балки) справа, зона сбора (яма) между ними, и трос с **процедурно сгенерированной текстурой каната**. Tap по светящейся точке зацепа → машина натягивает трос → конструкция рушится → кирпичи падают, попавшие в зону считаются в HUD.

## 2. Что изменилось с прошлой итерации
- Game-сцена больше не заглушка: удалён "WIP"-текст и старый Ground_Placeholder.
- Камера перенастроена под кадр со зданием (позиция Y=2.5, ortho size 10).
- HUD-счётчик добавлен в уже существующий Canvas/SafeArea из итерации 1.
- Менеджеры из итерации 1 не тронуты.

## 3. Editor скрипты: что запустить и в каком порядке
1. Скопируй обновлённую папку `Assets/BrickGame` в проект (поверх итерации 1 — файлы итерации 1 нужны).
2. Дождись компиляции (нужны DOTween free и TMP Essentials).
3. В меню Unity: **BrickGame → Setup Iteration 2 (Demolition Scene)**

Скрипт сам генерит текстуры (square.png, rope.png), материал троса, physics material, наполняет сцену `Game` и прокидывает все ссылки. Запускать можно повторно — старые объекты пересоздаются без дублей.

> Если итерация 1 ещё не настроена в этом проекте — сначала запусти **Setup Iteration 1**, затем Iteration 2 (HUD добавляется в Canvas из ит.1).

## 4. Как тестировать
1. Открой `Assets/BrickGame/Scenes/Game.unity`.
2. Game-вью в портрет (1080x1920).
3. Нажми Play.
4. Тапай по разным оранжевым кругам (точкам зацепа) на здании.

## 5. Ожидаемый результат
- Слева синяя машина с кабиной, справа высокая башня (~42 кирпича): фиолетовые опоры внизу, синие кирпичи, оранжевые балки.
- Между ними — размеченная зона сбора (оранжевый пол + два столба).
- На здании пульсируют 4 точки зацепа (низ левой опоры, середина, верх, низ правой опоры).
- Tap по точке → точка делает punch, машина дёргается назад, трос с текстурой каната натягивается от машины к кирпичу, кирпич выдёргивается → башня рушится реальной физикой.
- Кирпичи сталкиваются, катятся, часть падает в зону сбора.
- Счётчик "Bricks: X / 42" вверху обновляется с анимацией по мере попадания кирпичей в зону.
- Можно тапнуть другую точку и дотянуть ещё (несколько бросков).

**Подсказка по геймплею:** тяга за нижнюю опору роняет башню влево в яму (хороший результат), тяга за верх разбрасывает кирпичи (плохой) — это и есть головоломка из ТЗ.

## 6. Известные ограничения текущей итерации
- Нет попапа результата и звёзд, нет кнопки "В бой" (итерация 3).
- Счётчик "живой" (считает кирпичи, находящиеся в зоне сейчас) — может слегка дёргаться во время хаоса, итоговый замер по оседанию будет в ит.3.
- Нет частиц, пыли, screen shake, звуков ударов кирпичей (итерация 4) — пока только tween-анимации троса/машины/точек.
- Здание одно, фиксированное (система уровней — ит.6).
- Точки зацепа "приклеены" к своим кирпичам и следуют за ними.

## 7. Что будет в следующей итерации
Подсчёт по оседанию + попап результата со звёздами (1-3) + повторные броски + кнопка перехода в боевую фазу.

### Тех. детали
- Кирпичи: Dynamic Rigidbody2D + BoxCollider2D, gravityScale 1.6, общий PhysicsMaterial2D (friction 0.6, bounciness 0).
- Трос: LineRenderer, textureMode Tile, материал на Sprites/Default с сгенерированной tileable-текстурой каната.
- Тяга: ramp-force к выбранному кирпичу (Rope.maxPullForce / rampTime — можно крутить в инспекторе).
- Инпут блокируется над UI (IsPointerOverUI).
- Скрипты без namespace и без комментариев.
