# BrickGame — Iteration 04: Juice (полировка разрушения)

## 1. Что было добавлено
Сочность разрушения: облачка пыли при ударах кирпичей, тряска камеры на сильных обрушениях, звук удара (с вариацией высоты, без спама) и squash&stretch зоны сбора при попадании кирпича.

## 2. Что изменилось с прошлой итерации
- `Brick` — добавлен `OnCollisionEnter2D` (выше порога скорости + кулдаун на кирпич) → дёргает `ImpactManager`. Тюнинг физики и `Highlight` сохранены.
- `CollectionPit` — добавлено событие `OnBrickEntered(pos)` (для PitJuice). Логика подсчёта не тронута.
- `SoundManager` — добавлена перегрузка `PlaySfx(clip, pitch, volume)` + сброс pitch/volume в базовом методе. Остальное без изменений.
- Сцена разрушения и UI результата из ит.2-3 не пересобираются — editor ит.4 только добавляет juice-объекты.

## 3. Editor скрипты: что запустить и в каком порядке
1. Скопируй папку `Assets/BrickGame` поверх проекта (заменит `Brick.cs`, `CollectionPit.cs`, `SoundManager.cs`, добавит новые).
2. Дождись компиляции (DOTween free + TMP Essentials).
3. В меню Unity: **BrickGame → Setup Iteration 4 (Juice)**

> Требуется уже настроенная ит.2 (скрипт ищет Main Camera, CollectionPit, PitVisual). Если нет — warning «Run Iteration 2 first».

Скрипт генерит текстуру пылинки, материал пыли, звук удара, собирает prefab dust-партикла, добавляет `CameraShake` на камеру, `ImpactManager` (пул пыли) и `PitJuice` на зону. Идемпотентен.

## 4. Как тестировать
1. Открой `Assets/BrickGame/Scenes/Game.unity`, Play.
2. Ломай башню — следи за пылью, звуком, тряской.
3. Кидай кирпичи в зону сбора — пол и столбы должны пульсировать.

## 5. Ожидаемый результат
- При каждом заметном ударе кирпича — облачко пыли в точке контакта.
- Удары сопровождаются звуком с лёгкой вариацией высоты; при массовом обрушении звук не превращается в кашу (троттлинг).
- Сильные удары/обрушения трясут камеру (сила зависит от скорости удара).
- Когда кирпич попадает в зону сбора — пол и два столба зоны делают squash&stretch.

## 6. Известные ограничения текущей итерации
- На сами Rigidbody2D-кирпичи squash&stretch не вешается (ломает физику) — для них только пыль/звук/подсветка.
- Кнопка BATTLE всё ещё заглушка (бой — ит.5).
- Один фиксированный уровень (ит.6).
- Звук удара — синтезированный noise-плейсхолдер.

## 7. Что будет в следующей итерации
Боевая фаза (авто-бой): собранные кирпичи = N атак, волна врагов, авто-стрельба снарядами-кирпичами, победа/поражение, переход на следующий уровень.

---

### Тех. детали
- Пыль: ParticleSystem prefab (burst 10, gravity, fade, sizeOverLifetime), пул на 12 шт в `ImpactManager`.
- Тряска: `CameraShake.Shake()` через `DOShakePosition`, троттлинг по `shakeInterval`/`shakeThreshold`.
- Звук: троттлинг `soundInterval`, рандомный pitch 0.88-1.12, громкость по силе удара.
- Все пороги (`shakeThreshold`, `soundInterval` и т.д.) — в инспекторе `ImpactManager`.
- Скрипты без namespace и без комментариев.
