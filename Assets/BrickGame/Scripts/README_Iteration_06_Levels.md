# BrickGame — Iteration 06: Levels + Progression

## 1. Что было добавлено
Система уровней. Башня и ямы теперь строятся **в рантайме** `LevelManager` по конфигу выбранного уровня (`GameSession.Level`), а не пекутся в сцену. 5 уровней с ростом сложности; волна врагов в бою масштабируется по уровню; в HUD появился индикатор «Level N».

## 2. Что изменилось с прошлой итерации
- Запечённые `Building` / `CollectionPit` / `PitVisual` удалены из сцены — их строит `LevelManager` при старте.
- `DemolitionController` упрощён: только выбор кирпича (тап → ближайший) и тяга. Подсчёт/HUD переехали в `LevelManager`.
- `LevelFlowController` берёт собранное/всего из `LevelManager` (+ агрегирует несколько ям).
- `GameHUDController` — метод `SetLevel` + поле `levelText`.
- `PitJuice` — базовый масштаб теперь ловится в `Start` (нужно для рантайм-сборки ям).
- `BattleController` — волна масштабируется по `GameSession.Level` (видимость целей из прошлой правки сохранена).
- Тюнинг физики/выбора кирпича из ит.2 сохранён 1:1 (те же масса/гравитация/трение/тяга).

> «Ложные точки зацепа» из ТЗ не реализуются: точек больше нет (тап в любое место → ближайший кирпич). Сложность растёт высотой, числом колонн, тяжёлыми блоками и второй ямой.

## 3. Editor скрипты: что запустить
1. Скопируй папку `Assets/BrickGame` поверх проекта (заменит `DemolitionController`, `LevelFlowController`, `GameHUDController`, `PitJuice`, `BattleController`; добавит `LevelManager`/`LevelDef`).
2. Дождись компиляции.
3. В меню Unity: **BrickGame → Setup Iteration 6 (Levels)**

> Требуются настроенные ит.2-5. Battle-сцену пересобирать не нужно — `BattleController` подхватит новый код сам.
>
> **ВАЖНО:** после ит.6 не запускай повторно Setup ит.2/ит.4 (они пекут башню/ямы в сцену и создадут дубликаты с рантайм-сборкой). Если случайно запустил — просто прогони ит.6 ещё раз (он чистит запечённое).

## 4. Как тестировать
1. Открой `Game.unity`, Play → уровень 1 (по умолчанию `GameSession.Level=1`).
2. Чтобы быстро глянуть другой уровень без боёв: на объекте `LevelManager` уровни лежат в списке `Levels`; либо пройди бой (VICTORY → NEXT повышает уровень).
3. Полный цикл: ломаешь → COLLECT → BATTLE → победа → NEXT → следующий уровень (башня выше/сложнее, волна больше).

## 5. Ожидаемый результат
- Сверху по центру «Level N», ниже «Bricks: X / Y».
- Уровень 1-2: башня 2 колонны. Уровень 3+: тяжёлые (тёмные) блоки. Уровень 4-5: 3 колонны и **две зоны сбора** (считаются обе). Камера авто-отъезжает под высоту башни.
- Бой: чем выше уровень — тем больше врагов и (через каждые 3 уровня) больше HP.
- VICTORY → NEXT повышает `GameSession.Level` и грузит следующий уровень; DEFEAT/RETRY перезапускает текущий.

## 6. Известные ограничения текущей итерации
- Уровней 5; после последнего конфиг клампится на последнем (дальше не растёт).
- Нет экрана выбора уровня и сохранения прогресса между запусками (прогресс живёт в `GameSession` до перезапуска приложения).
- Две ямы — это две зоны сбора у одной башни (не две отдельные башни).

## 7. Что будет в следующей итерации
Экономика и апгрейды: кирпичи как валюта (банк, сохранение), экран апгрейдов (сила тяги, длина троса, скорость натяжения, магнит ямы), анимация начисления награды.

### Тех. детали
- `LevelManager.Awake` строит ямы и башню, агрегирует подсчёт, подгоняет камеру; конфиг — список `LevelDef` (rows/columns/heavyEvery/pitCount) в инспекторе.
- Кирпичи строятся с тем же тюнингом, тяжёлые блоки — масса ×2 и тёмный цвет.
- Скрипты без namespace и без комментариев.
