# BrickGame — Iteration 14: Juice Layer + Brick Consumption

## 1. Что было добавлено
Финальный juice-слой плюс твой запрос — кирпичи после зачёта в яме уничтожаются:
- **Уничтожение собранных кирпичей** — попав в яму, кирпич засчитывается и сразу убирается (сжимается+исчезает), чтобы не мешать остальным.
- **Combo** — серия попаданий в яму подряд наращивает множитель (`Combo xN`), с лёгким нарастающим шейком.
- **Slow-mo** — короткое замедление на крупном обрушении (сильный удар).
- **«+1» попапы** — всплывают над ямой при попадании кирпича.
- **Конфетти + фанфары** — на 3 звезды в окне результата.

## 2. Что изменилось с прошлой итерации
- `CollectionPit` — счёт теперь стойкий счётчик (только растёт); пойманный кирпич `Consume`: физика off, коллайдер off, сжатие+фейд, `Destroy`. Двойной зачёт исключён.
- `ComboManager` (новый) — комбо-окно, UI `Combo xN`, «+1» попапы (мировой TextMeshPro), эскалация шейка.
- `SlowMoController` (новый) — `Time.timeScale` дип на realtime-таймере с кулдауном.
- `ImpactManager` — на удар ≥ `slowMoThreshold` дёргает slow-mo.
- `ResultPopup` — на 3 звезды: фанфары + конфетти (DIY UI-конфетти поверх попапа).
- `LevelManager` — триггер ямы опущен (центр y=1, высота 2): кирпич засчитывается, когда реально входит в яму, а не пролетая высоко сверху (заодно чуть больше скилла).
- Прочая логика/тюнинг сохранены.

## 3. Editor скрипты: что запустить
1. Скопируй папку `Assets/BrickGame` поверх проекта (заменит `CollectionPit`, `ImpactManager`, `LevelManager`, `ResultPopup`; добавит `ComboManager`, `SlowMoController`, editor ит.14).
2. Дождись компиляции.
3. В меню Unity: **BrickGame → Setup Iteration 14 (Juice Layer)**

Скрипт генерит фанфары, добавляет `Combo`-текст в HUD, объекты `ComboManager`/`SlowMoController`, прокидывает фанфары в `ResultPopup`. Battle-сцену не трогает.

## 4. Как тестировать
1. Любой уровень: роняй кирпичи в яму — каждый даёт «+1» и **исчезает** (не копится в яме).
2. Несколько попаданий подряд (в пределах ~1.3с) → вверху `Combo x2/x3...`, на больших комбо — лёгкий шейк.
3. Крупное обрушение (мощный рывок/бомба) → кратко срабатывает slow-mo.
4. Собери на 3 звезды → в окне результата фанфары + падающее конфетти.

## 5. Ожидаемый результат
- Яма не забивается: собранные кирпичи убираются, новые свободно залетают.
- Комбо растёт за серии, «+1» приятно всплывает.
- Slow-mo подчёркивает большие обвалы.
- 3 звезды ощущаются как праздник (конфетти+звук).

## 6. Известные ограничения / тюнинг
- Комбо-окно: `ComboManager.comboWindow` (1.3с).
- Slow-mo: `SlowMoController.scale/duration/cooldown`; порог срабатывания — `ImpactManager.slowMoThreshold` (11).
- Размер/скорость «+1» попапа — в `ComboManager.SpawnPopup` (fontSize/scale).
- Триггер ямы опущен — если кирпичи стали засчитываться хуже/легче, правь size/позицию в `LevelManager.BuildPit`.
- Конфетти — простые UI-прямоугольники (без текстуры), чтобы гарантированно рисоваться поверх UI.

## 7. Статус
Все 4 укрупнённые итерации из плана готовы (камера/материалы → прогресс/выбор → пазл-слой → juice). Опционально остаются: daily challenge, достижения, реальный Taptic, арт/звук-ассеты.

### Тех. детали
- Уничтожение: `body.simulated=false` + коллайдер off сразу (перестаёт мешать), затем сжатие/фейд и `Destroy` через 0.32с.
- «+1»: мировой `TextMeshPro`, анимация вверх + фейд через `DOTween.To` по `tmp.alpha` (без TMP-DOTween модуля).
- Конфетти: `Sprite.Create` из `Texture2D.whiteTexture`, падение+вращение+фейд под рутом попапа.
- Скрипты без namespace и без комментариев.
