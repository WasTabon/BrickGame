# BrickGame — Iteration 13: Puzzle Layer

## 1. Что было добавлено
Пазл-слой — три механики, делающие уровни головоломками:
- **Движущаяся яма** — зона сбора ездит влево-вправо (нужно ловить момент броска).
- **Лимит бросков** — N тросов на уровень; счётчик в HUD; кончились → авто-COLLECT.
- **Хрупкие / липкие блоки** — хрупкий разбивается при сильном ударе (не засчитывается), липкий склеен с нижним блоком (стопку труднее свалить).

## 2. Что изменилось с прошлой итерации
- `LevelDef` — новые поля: `maxThrows`, `pitMoveAmplitude`/`pitMoveSpeed`, `brittleEvery`, `stickyEvery`.
- `PitMover` (новый) — синусоидальное движение ямы (визуал + триггер двигаются вместе).
- `Brick` — материалы `Brittle` (раскалывается = `Destroy`) и `Sticky` (склейка через `FixedJoint2D`).
- `LevelManager` — строит движущуюся яму, ставит стыковочные джоинты для липких, прокидывает `maxThrows` в `DemolitionController` и счётчик в HUD; ссылка `demolition`.
- `DemolitionController` — считает броски, обновляет HUD, при исчерпании блокирует ввод и вызывает авто-COLLECT.
- `GameHUDController` — счётчик бросков (`Throws: N`), скрыт если лимита нет.
- `LevelFlowController` — публичный `ForceCollect()` для авто-сбора.
- 15 уровней пересобраны с раскладкой новых механик (см. ниже).

**Поведение материалов:**
- 🩶 **Хрупкий** (бежевый) — при ударе быстрее порога раскалывается и исчезает (в яму не попадёт). Меньше «зачёта», но эффектно расчищает.
- 🟢 **Липкий** (зелёный) — приклеен к блоку под ним; рвётся только сильной тягой (joint breakForce 250). Стопку труднее обвалить.

## 3. Editor скрипты: что запустить
1. Скопируй папку `Assets/BrickGame` поверх проекта (заменит `LevelManager`, `LevelDef`, `Brick`, `DemolitionController`, `GameHUDController`, `LevelFlowController`; добавит `PitMover`, editor ит.13).
2. Дождись компиляции.
3. В меню Unity: **BrickGame → Setup Iteration 13 (Puzzle Layer)**

Скрипт добавляет счётчик бросков в HUD, прокидывает ссылки (`LevelManager.demolition`, `DemolitionController.hud/flow`) и пересобирает 15 уровней с механиками. Battle-сцену не трогает.

## 4. Раскладка механик по уровням (стартовая)
- Лимит бросков: ур. 4, 10, 12, 14, 15.
- Движущаяся яма: ур. 5, 9, 12, 15.
- Хрупкие: ур. 7, 13, 15.
- Липкие: ур. 8, 11, 14, 15.

## 5. Как тестировать
1. Открой карту, выбери нужный уровень (или меняй `GameSession.Level`).
2. Ур.4 — вверху появляется `Throws: 6`, каждый бросок уменьшает; на нуле — автоматически срабатывает COLLECT.
3. Ур.5 — яма ездит влево-вправо; лови момент.
4. Ур.7 — бежевые хрупкие блоки разлетаются при сильном падении (в зачёт не идут).
5. Ур.8 — зелёные липкие держат стопку; нужна сильная тяга, чтобы оторвать.

## 6. Известные ограничения текущей итерации
- Хрупкий блок при расколе просто исчезает (без осколков-партиклов) + пыль удара. Можно позже добавить разлёт фрагментов.
- Липкость = `FixedJoint2D` с порогом разрыва 250 (поле в коде `BuildTower`); если рвётся слишком легко/трудно — поменяй `breakForce/breakTorque`.
- Движение ямы общее для обеих ям (в фазе). Амплитуда/скорость — в `LevelDef`.
- Все числа уровней стартовые — крути в `LevelManager.levels` (инспектор) или в `BuildLevels()` редактора.

## 7. Что будет в следующей итерации
Juice-слой (финальный по плану): combo-множитель за серию попаданий, slow-mo на крупное обрушение, всплывающие «+1» при попадании в яму, конфетти + фанфары на 3 звезды.

### Тех. детали
- Движущаяся яма: `PitGroup` (визуал + триггер) колеблется по синусу через `PitMover` (Time.time → стоит на паузе).
- Лимит бросков: счёт в `DemolitionController`; авто-COLLECT через `LevelFlowController.ForceCollect()`.
- Липкость: `FixedJoint2D` к блоку ниже по колонке (после балки — к балке).
- Хрупкость: `Destroy` при ударе быстрее `shatterSpeed`.
- Скрипты без namespace и без комментариев.
