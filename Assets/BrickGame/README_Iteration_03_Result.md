# BrickGame — Iteration 03: Result + Stars

## 1. Что было добавлено
Завершение уровня: кнопка **COLLECT** в HUD, подсчёт собранного по оседанию и попап результата со звёздами (1-3), кнопками **RETRY** и **BATTLE**. Звезда — процедурно сгенерированный спрайт.

## 2. Что изменилось с прошлой итерации
- `GameHUDController` получил кнопку Collect (поле `collectButton` + анимация показа/скрытия). Живой счётчик не тронут.
- Сцена разрушения, тюнинг физики и логика выбора кирпича из ит.2 **не трогаются** — editor-скрипт ит.3 только добавляет UI в существующий Canvas.
- Управление инпутом: при нажатии Collect `DemolitionController` отключается (`enabled = false`), дальнейшие броски блокируются.

## 3. Editor скрипты: что запустить и в каком порядке
1. Скопируй папку `Assets/BrickGame` поверх проекта (заменит `GameHUDController.cs`, добавит новые скрипты).
2. Дождись компиляции (DOTween free + TMP Essentials).
3. В меню Unity: **BrickGame → Setup Iteration 3 (Result + Stars)**

> Перед этим в проекте уже должна быть настроена ит.2 (скрипт ищет существующие `Canvas`, `GameHUD`, `CollectionPit`, `DemolitionController`). Если их нет — будет warning «Run Setup Iteration 2 first».

Скрипт генерит `star.png`, добавляет Collect-кнопку и скрытый ResultPopup, вешает `LevelFlowController` и прокидывает ссылки. Идемпотентен (повторный запуск пересоздаёт без дублей). Сцену разрушения не пересобирает.

## 4. Как тестировать
1. Открой `Assets/BrickGame/Scenes/Game.unity`.
2. Game-вью в портрет (1080x1920), нажми Play.
3. Поломай башню (можно несколькими бросками).
4. Нажми **COLLECT** внизу.
5. На попапе попробуй **RETRY** и **BATTLE**.

## 5. Ожидаемый результат
- Внизу по центру — крупная кнопка COLLECT.
- После нажатия: кнопка прячется (scale-out), броски блокируются, ~0.6с на оседание.
- Выезжает попап: затемняющий backdrop (alpha 0→0.6), панель (scale 0→1, OutBack), заголовок RESULT, три звезды с поочерёдным pop-in (заработанные — оранжевые, остальные — тусклые), число «X / Y».
- Звёзды: ≥90% → 3, ≥70% → 2, ≥50% → 1.
- **RETRY** — плавный fade-перезаход на свежую башню.
- **BATTLE** — пока тоже перезапускает уровень (заглушка; реальный бой в ит.5).

## 6. Известные ограничения текущей итерации
- Кнопка BATTLE — заглушка (перезапуск уровня), боевая фаза в ит.5.
- Нет частиц/звуков на звёзды и удары, нет screen shake (ит.4).
- Один фиксированный уровень (система уровней — ит.6).
- Collect можно нажать в любой момент (точный «момент готовности» не требуется по ТЗ).

## 7. Что будет в следующей итерации
Полировка разрушения: пыль/частицы при ударах, screen shake, звук на каждый удар кирпича, squash & stretch, подсветка зоны сбора.

### Тех. детали
- Звезда: 5-конечный полигон, растеризация point-in-polygon в editor, спрайт с alphaIsTransparency.
- Попап: backdrop CanvasGroup (blocksRaycasts), панель/звёзды на DOTween (OutBack/InBack).
- Поток: `LevelFlowController` (Collect → settle → подсчёт → попап → Retry/Battle через TransitionManager).
- Скрипты без namespace и без комментариев.
